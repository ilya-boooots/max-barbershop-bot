# ONBOARDING BUSINESS (Owner in Telegram) 🚀

Ниже — быстрый сценарий запуска нового барбершопа без кода.

## 1) Роли в проекте 👥
- **Developer** — технический владелец, деплой, доступ к `/status` и диагностике.
- **Admin** — владелец/управляющий точкой, управляет операционкой.
- **Manager** — администратор смены, работает с записями и клиентами.

## 2) Первичный вход owner/admin ✅
1. Откройте бота.
2. Нажмите `/start`.
3. Проверьте, что отображается главное меню с кнопками и эмодзи.

## 3) Настройка ролей (copy-paste flow) 🛠️
1. Нажмите `👥 Персонал`.
2. Назначьте роли сотрудникам:
   - owner/директор → **admin**
   - старший админ → **manager**
   - разработчик → **developer**
3. Перезапустите сессию (или снова `/start`) и проверьте меню по роли.

## 4) Подключение YClients через UI 🔌
1. Нажмите `⚙️ Интеграция YClients`.
2. Заполните поля:
   - `Company ID`
   - `Partner Token`
   - `User Token`
   - `Base URL` (если нужен нестандартный)
3. Нажмите `🔌 Проверить YClients`.
4. Убедитесь, что видите `✅ YClients подключен...`.

## 5) Проверка записи end-to-end ✂️
1. Пользователь: `✂️ Записаться`.
2. Пройдите шаги: категория → услуга → мастер → дата → время.
3. Подтвердите запись.
4. Откройте `📅 Мои записи` и убедитесь, что запись есть.
5. (Если доступно) протестируйте отмену/перенос.

## 6) Проверка оператора 📋
1. Под admin/manager откройте `📋 Записи`.
2. Найдите созданную запись.
3. Откройте карточку, проверьте действия и возврат `⬅️ Назад`.

## 7) Support screen policy 🆘
- Экран `🆘 Поддержка` должен быть простым и статичным.
- Контакт берется из `SUPPORT_CONTACT` (например, `@your_support`).

## 8) Готово к запуску 🎉
- Роли назначены.
- YClients проверен.
- Тестовая запись прошла полный цикл.
- Навигация `⬅️ Назад` и `🏠 Главное меню` работает на всех ключевых экранах.
