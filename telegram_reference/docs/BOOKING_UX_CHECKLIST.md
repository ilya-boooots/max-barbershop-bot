# Booking UX checklist (YClients)

1. Категории: в списке нет `Без группы`, отображаются только непустые категории, экран показывается с фото барбершопа.
2. Услуги: длительность отображается в минутах (без `3600 мин` для часовой услуги).
3. Мастер: после выбора мастера отображается фото мастера (если доступно в API).
4. Даты: показываются только даты с доступными слотами выбранного мастера.
5. Время и создание записи:
   - показываются реальные слоты на выбранную дату;
   - успешное создание записи проходит без общего `Что-то пошло не так`;
   - при занятом слоте пользователь получает понятное сообщение и видит обновлённые слоты;
   - в dev-диагностику уходят trace_id/endpoint/status/snippet при ошибке.
