# Аудит навигации `⬅️ Назад` / `🏠 Главное меню`

Дата: 2026-05-27

## Методика
- Поиск всех inline/reply кнопок с текстами `⬅️ Назад`, `🏠 Главное меню`, а также алиасов `nav:back`, `nav:home`.
- Проверка, что `🏠 Главное меню` ведёт на единый глобальный callback `nav:home`.
- Проверка, что `⬅️ Назад` на публичных экранах не уводит в legacy-карты/мёртвые экраны.
- Проверка ограничений callback_data (тип строка, длина <= 64 байт).

## Глобальные стандарты
- Глобальный home callback: `nav:home` (`app/ui/callbacks.py`).
- Глобальный back callback: `nav:back` (`app/ui/callbacks.py`).
- Глобальные обработчики: `app/handlers/navigation.py` (`handle_nav_home`, `handle_nav_back`).
- Очистка FSM/стека при home и role-aware рендер выполняется через `app/core/navigation.py::home_handler`.

## Проблемы и фиксы в этом проходе

### Найдено
1. Не было единого регрессионного теста, который проверяет, что **все** кнопки `🏠 Главное меню` в коде завязаны на глобальный `nav:home`.
2. Не было отдельного smoke-теста, фиксирующего отсутствие legacy callback `contacts:maps` в публичной контактной клавиатуре.

### Исправлено
- Добавлен `tests/unit/test_navigation_back_home_consistency.py`:
  - проверка глобального home callback для всех строк с `🏠 Главное меню`;
  - проверка, что public contacts back не ссылается на `contacts:maps`;
  - проверка стабильности `nav:home`/`nav:back` и лимита длины callback_data.

## Остаточные риски (manual)
- Глубокие FSM-сценарии с редкими stale payload (например, удаление сущности в соседней сессии) требуют ручной валидации UX fallback.
- Разделы с профильным back callback (automation modules, master photos, роли) требуют ручной проверки «ровно один шаг назад» на живом диалоге.
