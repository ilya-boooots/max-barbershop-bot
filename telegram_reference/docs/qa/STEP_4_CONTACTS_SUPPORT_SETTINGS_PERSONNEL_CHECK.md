- Разделы, завязанные на внешние API/данные мастеров (например, редактирование фото мастеров), если при ручной проверке на стенде нет валидной YClients-конфигурации или данных.

## Чеклист проверки branch timezone
- Контакты: убедиться, что отображаемый режим работы филиала корректен и человекочитаем.
- Настройки рассылок/уведомлений: проверить, что quiet hours и время отображаются в timezone филиала.
- Персонал:
  - история ролей (`changed_at`) в timezone филиала;
  - журнал действий (`created_at`) в timezone филиала;
  - дата назначения роли (`role_assigned_at`) в timezone филиала.
- Проверить fallback: при отсутствии timezone филиала отображается безопасный fallback и есть warning в логах.

## Ручные шаги (не отмечены как PASS автоматически)

### MT-010 Контакты
1. Открыть `📍 Контакты`.
2. Проверить адрес/телефон/режим работы на отсутствие `None`, JSON, пустых техполей.
3. Проверить сценарий без данных YClients (или с пустыми полями): должен быть чистый fallback.
4. При наличии override в настройках контактов — проверить, что он применяется.

### MT-011 Поддержка
1. Открыть `🆘 Поддержка`.
2. Нажать кнопку `🆘 Написать в поддержку` — должна открываться ссылка `t.me/...`.
3. Проверить наличие `⬅️ Назад` и `🏠 Главное меню`.
4. Для роли без прав проверить запрет на редактирование поддержки через callback.

### MT-012 Настройки
1. Открыть `⚙️ Настройки`.
2. Пройти все видимые кнопки и убедиться, что у каждой есть handler и корректная навигация назад/домой.
3. Для редактируемых полей проверить сохранение и обработку невалидного ввода.
4. Проверить, что обычный пользователь не может зайти в админ-подразделы через прямой callback.

### MT-013 Персонал / роли
1. Открыть `👥 Персонал` под manager/developer.
2. Проверить список сотрудников, карточку, историю ролей, журнал действий.
3. Назначить роль допустимому пользователю, затем снять роль.
4. Проверить, что у ID `378881880` роль снять нельзя.
5. Проверить, что обычный пользователь не получает доступ к разделу через прямой callback.
