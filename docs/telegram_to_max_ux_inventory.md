## Notes for future Codex PRs

* Do not invent UX.
* Inspect Telegram reference first.
* Copy texts/buttons/order 1:1 where possible.
* One flow/one bug/one handler per PR.
* If Telegram behavior cannot be found, stop and report instead of guessing.
* After each parity PR, update this inventory file.
